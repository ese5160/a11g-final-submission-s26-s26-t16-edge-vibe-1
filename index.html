Project Summary

The device combines embedded sensing, custom hardware, user input, and cloud monitoring to create a responsive spelling practice experience.

01

Problem

Spelling practice can be passive and slow to correct. Edge Vibe makes each attempt interactive with immediate visual, audio, and dashboard feedback.

02

Solution

Users build words with letter cubes and buttons while the MCU validates input, updates the OLED display, and reports system state to Node-RED.

03

Internet Augmentation

The dashboard displays target words, user input, system status, ambient light alerts, and logged spelling errors in real time.

System

The system uses a Wi-Fi enabled SiWG917Y MCU, OLED display, ambient light sensor, magnetic buzzer, letter cubes, buttons, and a cloud-hosted Node-RED interface.

Device Description

Edge Vibe is an interactive spelling and learning system designed to help users practice word recognition and spelling through visual and audio feedback.


It integrates sensors, display, cloud connectivity, custom PCB hardware, and a Node-RED dashboard to create a modern educational experience.

Device Functionality

Users interact through letter cubes, physical buttons, and the cloud interface. The device displays target words, tracks user input, validates spelling, provides correctness feedback, and logs errors for later analysis.

Requirements

Final hardware and software requirement results for the completed prototype.

Hardware Requirements

  • HRS-01: The SiWG917Y MCU successfully handled system control and Wi-Fi communication.
  • HRS-02: The system supports 16 letters instead of 26 due to GPIO limitations and multiplexer constraints.
  • HRS-03: Digital multiplexer and letter decoding worked reliably for the supported inputs.
  • HRS-04: The ambient light sensor functioned correctly and data was displayed on the dashboard with low-light alerts.
  • HRS-05: OLED display worked, showing simple animation according to the user input.
  • HRS-06: The magnetic buzzer provided reliable audio feedback for system events.
  • HRS-07: Base lighting was not implemented and was replaced with a dashboard-based feedback mechanism.
  • HRS-08: Control buttons functioned correctly after redesign and simplification.

Software Requirements

  • SRS-01: Letter tile detection latency met the ≤100 ms requirement during testing.
  • SRS-02: Letter encoding and recognition worked correctly for the supported 16-letter subset.
  • SRS-03: The system correctly validated user input against the target word.
  • SRS-04: Feedback was provided within the required 500 ms timeframe.
  • SRS-05: The OLED display reliably showed target words and system status.
  • SRS-06: Wi-Fi communication with the Node-RED dashboard worked successfully for data exchange.
